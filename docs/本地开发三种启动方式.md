## 🚀 本地开发三种启动方式

### 方式 1：一键脚本（推荐，最省心）

```
cd stamp-ai-service
chmod +x dev.sh   # 首次只需一次
./dev.sh
```

**自动完成**：

- 创建/复用 `.venv` 虚拟环境
- 安装 `requirements.txt` 依赖
- 创建 `data/` 目录
- **预下载 `u2net.onnx` 模型**（避免首次请求等待）
- 启动 `uvicorn --reload` 热重载

访问：http://127.0.0.1:18080/docs

------

### 方式 2：手动命令（适合调试）

```
cd stamp-ai-service

# 1. 复制配置（首次）
cp .env.example .env

# 2. 创建虚拟环境 + 装依赖
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 3. 启动（热重载）
uvicorn app:app --reload --port 18080
```

------

### 方式 3：IDE 直接 Run（PyCharm / VS Code）

- **PyCharm**：`Run → Edit Configurations → Environment variables` 勾选 `EnvFile` 选 `.env`
- **VS Code**：`.vscode/launch.json` 加 `"envFile": "${workspaceFolder}/.env"`

## 🐳 Docker/生产环境**完全不变**

| 环境               | 配置来源                                            | 生效方式                                           |
| ------------------ | --------------------------------------------------- | -------------------------------------------------- |
| **本地开发**       | 项目根目录 `.env`                                   | `load_dotenv()` 自动读取                           |
| **Docker Compose** | `docker-compose.yml` 的 `environment:` / `env_file` | 容器启动自动注入环境变量，`load_dotenv()` 静默跳过 |
| **生产裸机**       | `/etc/stamp-ai/.env` 或 systemd `EnvironmentFile`   | systemd 启动前注入                                 |

> **关键点**：`app.py` 里 `try/except ImportError` 保护，**生产环境没装 `python-dotenv` 也不报错**，`load_dotenv()` 找不到文件直接返回 `False`。

## 🔧 下次改配置只需：

```bash
# 1. 改 .env
vim .env

# 2. 本地：重启服务（Ctrl+C 再跑 dev.sh）
#    Docker: docker compose up -d --no-deps stamp-ai  (5秒)
```
