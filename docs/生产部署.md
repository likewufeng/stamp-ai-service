### ✅ 必须上传（核心运行文件）

```bash
stamp-ai-service/
├── app.py                 # 入口
├── config.py              # 配置（已环境变量化）
├── requirements.txt       # Python 依赖
├── Dockerfile             # 镜像构建
├── docker-compose.yml     # 编排
├── .env.example           # 配置模板（生产按此创建 .env）
├── deploy/
│   ├── deploy.sh          # 一键部署脚本
│   └── nginx/             # 如用 Nginx 反代
├── api/                   # 路由
├── core/                  # 核心业务逻辑
├── schemas/               # Pydantic 模型
├── utils/                 # 工具模块
└── .dockerignore          # 构建忽略（很重要！）
```

### ❌ 不上传（本地生成/敏感/大体积）

| 文件/目录                | 原因                                                         |
| ------------------------ | ------------------------------------------------------------ |
| `.env`                   | **含敏感配置**，生产现场创建                                 |
| `.venv/` / `venv/`       | 虚拟环境，Docker 构建时重装                                  |
| `data/`                  | 运行时数据目录（上传、输出、日志、模型），`deploy.sh` 会自动创建 |
| `__pycache__/` / `*.pyc` | 字节码缓存                                                   |
| `.git/`                  | 版本历史（生产不需要）                                       |
| `.idea/` / `.vscode/`    | IDE 配置                                                     |
| `*.log`                  | 本地日志                                                     |
| `*.onnx`                 | 模型文件（构建时下载/挂载）                                  |

### 🚀 两种上传方式

#### 方式 1：Git 部署（推荐，版本可控）

Bash



```bash
# 本地：提交代码（.gitignore 已排除上述 ❌ 文件）
git add .
git commit -m "release v0.6.0"
git push

# 生产服务器：
git clone <repo-url> stamp-ai-service
cd stamp-ai-service
cp .env.example .env
vim .env   # 必改：OUTPUT_URL_PREFIX=https://你的域名/outputs
./deploy/deploy.sh
```

#### 方式 2：打包传输（无 Git 环境）

```bash
# 本地：打包（排除无用文件）
tar -czf stamp-ai-service.tar.gz \
  --exclude='.venv' --exclude='venv' --exclude='data' \
  --exclude='__pycache__' --exclude='.git' --exclude='.idea' \
  --exclude='*.log' --exclude='.env' \
  stamp-ai-service/

# 传到服务器（scp/rsync/网盘）
scp stamp-ai-service.tar.gz user@server:/opt/

# 服务器解压部署
tar -xzf stamp-ai-service.tar.gz -C /opt/
cd /opt/stamp-ai-service
cp .env.example .env
vim .env
./deploy/deploy.sh
```

### 🔐 生产 `.env` 必改项（首次部署）

```bash
cp .env.example .env
vim .env

# 必改：
OUTPUT_URL_PREFIX=https://api.yourdomain.com/outputs   # 公网访问地址
# 可选改：
DATA_ROOT=/data/stamp-ai              # 挂载大盘/NAS
MAX_UPLOAD_BYTES=52428800             # 50MB
```

## 修正.env里配置方法

```
cd /opt/stamp-ai-service

# 1. 拉取最新 compose（或手动编辑 docker-compose.yml）
git pull

# 2. 修正 .env 里的 OUTPUT_URL_PREFIX（去掉 markdown 语法）
sed -i 's|^OUTPUT_URL_PREFIX=.*|OUTPUT_URL_PREFIX=http://192.168.9.167:18080/outputs|' .env

# 3. 验证
grep OUTPUT_URL_PREFIX .env
# 应输出：OUTPUT_URL_PREFIX=http://192.168.9.167:18080/outputs

# 4. 重启服务（带上新环境变量）
docker compose up -d --no-deps stamp-ai

# 5. 验证接口返回
curl -s http://127.0.0.1:18080/api/health
curl -X POST "http://192.168.9.167:18080/api/stamp/extract" \
  -F "file=@test.jpg" \
  -F "return_type=url" | jq .stamps[0].url
# 应返回：http://192.168.9.167:18080/outputs/<request_id>/stamp_001.png
```
