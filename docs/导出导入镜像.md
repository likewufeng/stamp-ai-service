## 推荐做法：导出/导入镜像

### 在 Windows 上

```bash
# 先构建
docker compose build

# 看镜像名
docker images | grep stamp

# 导出（名字按你本地实际镜像名改）
docker save -o stamp-ai-service.tar stamp-ai-service-stamp-ai:latest

# 若 nginx 也要用本地镜像（一般 nginx 用官方镜像，Linux 上可直接 pull）
# docker save -o nginx.tar nginx:1.27-alpine
```

把 `stamp-ai-service.tar`（以及代码/compose/`.env`/`data`）拷到 Linux。

### 在 Linux 上

```bash
docker load -i stamp-ai-service.tar
docker images   # 确认镜像在

# 用项目里的 compose 启动（目录结构要带上）
cd stamp-ai-service
cp .env.example .env   # 按需改端口等
mkdir -p data/{uploads,outputs,logs,temp,models/u2net}
chmod -R 777 ./data
docker compose up -d
```

> 注意：`docker compose up -d` 若发现本地已有对应镜像，通常不会重新 build；
> 若 compose 里写了 `build:`，有时仍可能尝试 build。更稳的是：

```bash
docker compose up -d --no-build
```

或先 `docker tag` 成 compose 期望的名字。

```bash
docker compose up -d --no-deps stamp-ai
```
